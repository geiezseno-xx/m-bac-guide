# m-bac-guide

## 项目简介

本仓库用于归档和发布多个独立的 HTML 页面，主要作为通用页面资料的存储与展示空间。仓库内容不针对任何特定网站或域名，旨在提供一个结构化的页面集合，便于浏览、引用和维护。

## 目录说明

```
m-bac-guide/
├── index.html        # 入口页面（可自定义）
├── pages/            # 存放独立 HTML 页面的目录
│   ├── page-01.html
│   ├── page-02.html
│   └── ...
├── assets/           # 静态资源（CSS、JS、图片等）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md         # 本说明文件
```

- **pages/**：所有独立 HTML 页面均存放于此，每个文件为一个独立页面。
- **assets/**：存放页面所需的样式、脚本和图片等资源，便于统一管理。

## 页面归档说明

- 每个页面均为独立的 HTML 文件，可单独访问或通过入口页导航。
- 页面内容不涉及任何特定商业、营销或诱导性信息，仅作为资料归档用途。
- 归档的页面可能包含示例、说明、工具界面或其他纯信息性内容。
- 页面之间无强依赖关系，可独立更新或移除。

## 维护说明

- 欢迎提交 Issue 或 Pull Request 来补充、修正或优化页面内容。
- 新增页面请放置于 `pages/` 目录，并确保文件名简洁、具有可读性。
- 如涉及外部资源引用，建议优先使用相对路径或通用 CDN，避免硬编码特定域名。
- 本仓库不承担任何因页面内容引发的法律责任，请确保提交的内容合法合规。

## 使用方式

1. 克隆或下载本仓库。
2. 直接打开 `index.html` 或 `pages/` 下的任意 HTML 文件即可浏览。
3. 可通过 GitHub Pages 部署为在线站点（需自行配置）。
